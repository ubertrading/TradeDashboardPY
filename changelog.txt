Hedge Disbalance Detection Alert Implementation
The Hedge Disbalance Detection feature has been fully implemented into the Trading Dashboard, providing automated monitoring of hedge integrity across all connected accounts.

Changes Made
1. Configuration Settings
Added new configurable settings to the dashboard's persistent state:

disbalance_alert_enabled: Toggles the feature on/off.
disbalance_alert_period_sec: The confirmation period in seconds (default 30s) before an alert is triggered.
disbalance_alert_email: Toggles email delivery.
disbalance_alert_telegram: Toggles Telegram delivery.
2. Disbalance Detection Loop (_disbalance_alert_loop)
Introduced a dedicated background thread that runs every 2 seconds to perform the disbalance checks:

Global Net Lots Check: Sums the buy and sell volumes across all instruments and all connected accounts in ea_account_info. If the absolute net volume exceeds 0.001, a timer starts.
Confirmation Period: If the global disbalance persists for longer than the configured period, it triggers the subgroup analysis.
Subgroup Analysis:
Groups accounts by their hedge group identifier (e.g., HUGO-01).
Calculates the net lots specifically for each subgroup.
Determines the connection status of all expected accounts in the subgroup (checking EA heartbeats, FIX manager, and MT Direct connection status).
Alert Generation: Constructs an alert payload highlighting the subgroup, the calculated net lots, and a per-account breakdown of positions.
If any account in the subgroup is disconnected, the alert is prefixed with "Possible Disbalance Detected" and includes a warning message listing the disconnected accounts.
If all accounts are connected, it sends a standard "Disbalance Detected" alert.
Cooldown: Added a 5-minute cooldown per subgroup to prevent alert spam.
3. Frontend Dashboard Updates
Updated the Settings tab in the UI:

Added a new section: 🚨 Hedge Disbalance Alert.
Inserted toggles for Enable/Disable, Email, and Telegram.
Inserted a number input for adjusting the Confirmation Period (seconds).
Implemented the necessary JavaScript functions (toggleDisbalanceSuboptions, saveDisbalanceAlert, saveDisbalanceSuboption) to persist these settings via the /api/settings endpoint.
